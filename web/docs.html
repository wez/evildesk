Wez's Evil Shell
A Windows Desktop Replacement

You can find the latest and more complete version of this documentation online at the EvilDesk Web Site.

What is it?
A replacement for the default Windows®XP graphical shell (taskbar, start menu and "tray").
Why would I want one?

The default explorer shell uses quite a lot of run-time resources and can be quite slow to use for "Power Users". This shell provides the following features but uses only a fraction of the resources that explorer.exe uses:

  • Quicklaunch icon bar
  • System Notification area (aka "Tray"), including tray bubbles
  • Clock
  • No classic taskbar, but "flashing" windows have their icons bounced to get your attention
  • Light-weight icon-free start menu
  • Kludge-free Workspace management; use up to 4 virtual desktops (well, 32 is the technical limit, but only 4 can be accessed via the UI right now)
  • icon-free desktop

How do I use it?

First of all, you need to install it (use your computer administrator account to do this). Once installed, you need to turn it on for your account, reboot and log back in again.

Now that you're running with the shell, you should be able to do most of the things that you're used to doing with explorer.exe, although the desktop looks a little different.

The shell turns on the following hotkeys:

Win E
Opens up an explorer window for file management.
Win R
Opens up the "Run..." dialog for launching commands.
Alt F1
Alt F2
Alt F3
Alt F4
Switches to Workspace 1, 2, 3 or 4
Win W
Displays the Workspace menu for the current window. You may use this menu to move the window to an alternate workspace, make it appear on all workspaces and also toggle transparency of the window.
Alt TAB
Alt SHIFT TAB
Displays the task switching window. Alt-Tab moves forwards through the list, Alt-Shift-Tab moves backwards. The list of tasks is limited to the tasks on the current workspace. While Alt is held down, you may press Enter to cycle to the next workspace. When you release Alt, the highlighted task will be activated, switching the workspace if required.
